# 月次PDCAダッシュボード モック説明資料

## 本資料の目的
月次PDCA資料（PDF）を運用されていた方に、本ダッシュボードモックが「何を・どう見せているか」を説明し、データの正確性や表示仕様について確認いただくためのものです。

## 5. PDFにない付加価値（ダッシュボード固有の機能）

| 機能 | 説明 | PDFとの違い |
|------|------|-------------|
| **単月/累月ワンクリック切替** | 全パネル・全数値が即座に切り替わる | PDFでは別ページ or 別資料 |
| **ドリルダウン** | KPI・WF棒・ブランド・チャネルをクリックで深堀り | PDFではページ遷移が必要 |
| **ホバーツールチップ** | BU行にマウスを乗せると業績5指標がポップアップ | PDFでは不可 |
| **展開/折りたたみ** | P&Lやブランドの親子行を開閉 | PDFでは全行常時表示 |
| **セル内スパークライン** | ヒートマップの各セルに11ヶ月推移の折れ線 | PDFでは数値のみ |
| **5段階ヒートマップ着色** | 達成率に応じた自動色分け（≥105%濃緑 → <90%濃赤） | PDFでは白黒 or 手動色付け |
| **信号アイコン（●）** | 緑/黄/赤ドットで達成状況を即座に識別 | PDFでは数字を読む必要 |
| **△記号** | 負の差異値を△で表示（会計慣行に準拠） | 同等 |

## 6. 確認事項（PDF担当者への質問リスト）

### A. データの正確性

| # | 確認事項 | 背景 |
|---|---------|------|
| A1 | **KPI 5枚の数値・単位は正しいか？** 特に事業利益11.6億円（単月）/207.1億円（累月）、原材料コスト105百万円/1111百万円 | モックデータのため、実際の11月度値と照合が必要 |
| A2 | **変動費高比は「逆指標」（低いほど良い）として扱っているが正しいか？** 計画比は plan÷actual で表示 | 逆指標の定義が資料によって異なる可能性 |
| A3 | **ウォーターフォールの増減項目（数量差/単価差/変動費差/固定費差）の分解ロジックはPDFと一致しているか？** | P.37-38の分解と対応確認 |
| A4 | **連結領域別損益の13行の項目名・並び順はPDFと一致しているか？** | P.36の行構成と対応確認 |
| A5 | **ブランド名の表記ゆれはないか？**（例：「プラズマ乳酸菌」vs「iMUSE」の使い分け） | 同一ブランドが別名で登場するケースの確認 |

### B. 表示ロジック

| # | 確認事項 | 背景 |
|---|---------|------|
| B1 | **達成率の色分け閾値（105/100/95/90%）は実務の判断基準と合っているか？** | 社内基準と異なる場合は調整が必要 |
| B2 | **「累月」は1月〜当月累計（暦年基準）で正しいか？** 4月起算の年度累計ではないか？ | 会計年度の定義確認 |
| B3 | **コスト項目の「達成率の良し悪しが逆」（実績＜計画＝良い）は全コスト行に適用すべきか？** | 一部コスト項目は「使い切る方が良い」場合もある |
| B4 | **原材料コストKPIに目標値・達成率を表示しないのは意図通りか？** | 現在はニュートラル色（灰色）で表示 |

### C. 構成・導線

| # | 確認事項 | 背景 |
|---|---------|------|
| C1 | **ドリルダウン対象外のブランド（ファイア/小岩井/その他）はクリック無効でよいか？** 深堀データが未整備のため | 将来的にデータ追加の予定があるか |
| C2 | **BUはドリルダウンパネルなし（ホバーツールチップのみ）でよいか？** | 以前はBUドリルダウンがあったが簡略化済み |
| C3 | **市場パネルの競合5社（CCJC/SU/A/I/KBC）の選定と略称は正しいか？** | 社内で使われている略称と一致しているか |
| C4 | **キーメッセージ欄の内容は誰が/どのタイミングで更新する想定か？** | 現在は静的テキスト |
| C5 | **Tab 1（連結視点）とTab 2（単社視点）の使い分けは、現在のPDF資料の構成と整合しているか？** | 元資料のセクション分けとの対応確認 |

### D. 運用面

| # | 確認事項 | 背景 |
|---|---------|------|
| D1 | **データ更新頻度と更新タイミングは？**（月次締め後何営業日で更新されるか） | ダッシュボード化した場合の運用フロー設計 |
| D2 | **閲覧者の範囲は？**（経営層のみ / 部長以上 / 全社公開） | アクセス権限の設計に影響 |
| D3 | **PDF資料と本ダッシュボードは並行運用か、置き換えか？** | 移行計画の確認 |
| D4 | **スパークラインの11ヶ月トレンドデータは実績値から生成しているが、元資料にこの推移データは存在するか？** | 現在は最終達成率から逆算で生成（モック） |

## 補足：本モックの制約
- **静的データ**: 全数値はハードコードされたモックデータ（API接続なし）

- **1920×1080固定**: レスポンシブ非対応（モニタ/プロジェクタ専用）
